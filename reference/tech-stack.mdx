## Overview

Tendon is built with modern, production-ready technologies that prioritize developer experience, performance, and type safety. This page provides a complete reference of all dependencies and the reasoning behind each choice.

## Core Framework

<CardGroup cols={2}>
  <Card title="Next.js" icon="react" href="https://nextjs.org">
    **Version:** 16.1.6
    
    React framework with server-side rendering, static generation, and file-based routing.
    
    **Why we chose it:**
    - Industry-standard React framework
    - Excellent performance with React Server Components
    - Built-in routing and API routes
    - Optimal SEO and loading performance
  </Card>
  
  <Card title="React" icon="atom" href="https://react.dev">
    **Version:** 19.2.4
    
    JavaScript library for building user interfaces with components.
    
    **Why we chose it:**
    - Most popular UI library
    - Massive ecosystem and community
    - Server Components support
    - Excellent TypeScript integration
  </Card>
  
  <Card title="TypeScript" icon="code" href="https://www.typescriptlang.org">
    **Version:** 5.x
    
    Typed superset of JavaScript that compiles to plain JavaScript.
    
    **Why we chose it:**
    - Type safety prevents runtime errors
    - Superior developer experience with autocomplete
    - Better refactoring and maintainability
    - Industry standard for serious projects
  </Card>
  
  <Card title="Vinext" icon="bolt" href="https://vinext.org">
    **Version:** 0.0.14
    
    Vite-powered Next.js alternative for faster development and builds.
    
    **Why we chose it:**
    - Extremely fast HMR with Vite
    - Better development experience
    - Compatible with Next.js ecosystem
    - Faster builds than traditional Next.js
  </Card>
</CardGroup>

## Authentication

<Card title="Better Auth" icon="shield-halved" href="https://better-auth.com">
  **Version:** 1.4.20
  
  Modern, type-safe authentication library for Next.js.
  
  **Why we chose it:**
  - Built specifically for Next.js and React Server Components
  - Type-safe with excellent TypeScript support
  - Easy to customize and extend
  - Supports multiple auth strategies (email, OAuth, etc.)
  - No vendor lock-in - you own your database
  
  **Features included:**
  - Email/password authentication
  - Session management
  - User management
  - Secure cookie handling
</Card>

## Database

<CardGroup cols={2}>
  <Card title="Drizzle ORM" icon="database" href="https://orm.drizzle.team">
    **Version:** 0.45.1
    
    TypeScript ORM with SQL-like syntax and excellent type inference.
    
    **Why we chose it:**
    - Full type safety with TypeScript
    - SQL-like syntax (easy to learn)
    - Lightweight and performant
    - Great migration system
    - No magic - you control your queries
    
    **Dev Dependency:** `drizzle-kit` (0.31.9) for migrations
  </Card>
  
  <Card title="Turso / LibSQL" icon="server" href="https://turso.tech">
    **Version:** @libsql/client 0.17.0
    
    Edge-hosted SQLite-compatible database with global replication.
    
    **Why we chose it:**
    - SQLite compatibility (easy to develop locally)
    - Edge deployment for low latency
    - Generous free tier
    - Branching and point-in-time recovery
    - Works with Drizzle ORM perfectly
  </Card>
</CardGroup>

## UI Components & Styling

<CardGroup cols={2}>
  <Card title="Tailwind CSS" icon="paintbrush" href="https://tailwindcss.com">
    **Version:** 4.x
    
    Utility-first CSS framework for rapid UI development.
    
    **Why we chose it:**
    - Rapid development with utility classes
    - Consistent design system
    - Tiny production bundle (purged CSS)
    - Excellent documentation
    - Industry standard
    
    **Related packages:**
    - `@tailwindcss/postcss` (4.x) - PostCSS integration
    - `tw-animate-css` (1.4.0) - Animation utilities
  </Card>
  
  <Card title="shadcn/ui" icon="palette" href="https://ui.shadcn.com">
    **Version:** 3.8.5 (dev dependency)
    
    Copy-paste component library built on Radix UI and Tailwind.
    
    **Why we chose it:**
    - Own your components (not a dependency)
    - Beautiful, accessible components
    - Built on Radix UI primitives
    - Highly customizable
    - Active community
  </Card>
  
  <Card title="Radix UI" icon="circle-nodes" href="https://www.radix-ui.com">
    **Version:** 1.4.3
    
    Unstyled, accessible component primitives.
    
    **Why we chose it:**
    - Full accessibility (ARIA) support
    - Keyboard navigation built-in
    - Unstyled (style however you want)
    - Used by shadcn/ui
    
    **Related:** `@base-ui/react` (1.2.0) - Additional primitives
  </Card>
  
  <Card title="Lucide Icons" icon="icons" href="https://lucide.dev">
    **Version:** 0.575.0
    
    Beautiful, consistent icon library with React components.
    
    **Why we chose it:**
    - 1000+ icons
    - Tree-shakeable (only bundle what you use)
    - Consistent design
    - TypeScript support
    - Active maintenance
  </Card>
</CardGroup>

## Technology Decisions

### Why These Choices?

<AccordionGroup>
  <Accordion title="Modern Stack" icon="star">
    Every technology is actively maintained, has strong TypeScript support, and represents current best practices in React development.
  </Accordion>
  
  <Accordion title="Performance First" icon="rocket">
    From Vite's instant HMR to Drizzle's lightweight queries and edge-deployed Turso database, every choice optimizes for speed.
  </Accordion>
  
  <Accordion title="Type Safety" icon="shield">
    TypeScript throughout, Zod for runtime validation, Drizzle for type-safe queries, and Better Auth for type-safe authentication.
  </Accordion>
  
  <Accordion title="Developer Experience" icon="heart">
    Fast builds with Vite, excellent TypeScript autocomplete, hot reload, beautiful UI components, and comprehensive CLI tools.
  </Accordion>
  
  <Accordion title="No Vendor Lock-in" icon="unlock">
    You own your auth database (Better Auth), your UI components (shadcn), and your queries (Drizzle). Easy to migrate away if needed.
  </Accordion>
  
  <Accordion title="Production Ready" icon="check">
    All packages are battle-tested, have large communities, and are used by major companies in production.
  </Accordion>
</AccordionGroup>
