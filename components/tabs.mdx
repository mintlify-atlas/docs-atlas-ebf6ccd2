The Tabs component provides an accessible way to organize content into multiple panels, showing one panel at a time. Built on Radix UI's Tabs primitive with support for horizontal and vertical orientations.

## Installation

The Tabs component is included by default. To add it to a new project:

```bash
npx shadcn@latest add tabs
```

## Import

```tsx
import {
  Tabs,
  TabsList,
  TabsTrigger,
  TabsContent,
} from "@/components/ui/tabs"
```

## Basic Usage

```tsx
import { Tabs, TabsList, TabsTrigger, TabsContent } from "@/components/ui/tabs"

export default function Example() {
  return (
    <Tabs defaultValue="account">
      <TabsList>
        <TabsTrigger value="account">Account</TabsTrigger>
        <TabsTrigger value="password">Password</TabsTrigger>
      </TabsList>
      <TabsContent value="account">
        <p>Account settings content</p>
      </TabsContent>
      <TabsContent value="password">
        <p>Password settings content</p>
      </TabsContent>
    </Tabs>
  )
}
```

## Components

### Tabs

Root container component.

```tsx
<Tabs
  defaultValue="tab1"
  value={value}
  onValueChange={setValue}
  orientation="horizontal"
>
  {/* tabs content */}
</Tabs>
```

**Props:**
- `defaultValue` - Default active tab (uncontrolled)
- `value` - Active tab (controlled)
- `onValueChange` - Callback when tab changes
- `orientation` - "horizontal" | "vertical" (default: "horizontal")

### TabsList

Container for tab triggers.

```tsx
<TabsList variant="default">
  <TabsTrigger value="tab1">Tab 1</TabsTrigger>
  <TabsTrigger value="tab2">Tab 2</TabsTrigger>
</TabsList>
```

**Props:**
- `variant` - "default" | "line"

**Variants:**
- `default` - Pill-style tabs with background
- `line` - Underline-style tabs

### TabsTrigger

Individual tab button.

```tsx
<TabsTrigger value="overview">
  Overview
</TabsTrigger>
```

**Props:**
- `value` - Unique identifier for this tab
- `disabled` - Disable the tab

### TabsContent

Content panel for each tab.

```tsx
<TabsContent value="overview">
  <p>Overview content</p>
</TabsContent>
```

**Props:**
- `value` - Matches the trigger value

### Controlled Tabs

```tsx
"use client"

import { useState } from "react"
import { Tabs, TabsList, TabsTrigger, TabsContent } from "@/components/ui/tabs"
import { Button } from "@/components/ui/button"

export default function ControlledTabs() {
  const [activeTab, setActiveTab] = useState("tab1")

  return (
    <div>
      <Tabs value={activeTab} onValueChange={setActiveTab}>
        <TabsList>
          <TabsTrigger value="tab1">Tab 1</TabsTrigger>
          <TabsTrigger value="tab2">Tab 2</TabsTrigger>
          <TabsTrigger value="tab3">Tab 3</TabsTrigger>
        </TabsList>
        <TabsContent value="tab1">Content 1</TabsContent>
        <TabsContent value="tab2">Content 2</TabsContent>
        <TabsContent value="tab3">Content 3</TabsContent>
      </Tabs>
      <div className="mt-4 flex gap-2">
        <Button onClick={() => setActiveTab("tab1")}>Go to Tab 1</Button>
        <Button onClick={() => setActiveTab("tab2")}>Go to Tab 2</Button>
        <Button onClick={() => setActiveTab("tab3")}>Go to Tab 3</Button>
      </div>
    </div>
  )
}
```

## Accessibility Features

- **Keyboard navigation**: Arrow keys to navigate between tabs
- **Focus management**: Automatic focus handling
- **ARIA attributes**: Proper `role="tablist"`, `role="tab"`, `role="tabpanel"`
- **Selection**: Home/End keys to jump to first/last tab
- **Disabled state**: Proper disabled styling and behavior

## Best Practices

### Use Descriptive Labels

```tsx
{/* Good */}
<TabsTrigger value="overview">Overview</TabsTrigger>

{/* Avoid */}
<TabsTrigger value="tab1">Tab 1</TabsTrigger>
```

### Limit Number of Tabs

For better UX, limit to 5-7 tabs. For more options, consider a different navigation pattern.

### Keep Content Related

Only use tabs for closely related content that users might want to switch between.
