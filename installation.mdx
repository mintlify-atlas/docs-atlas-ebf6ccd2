## Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Node.js** - Version 20 or higher recommended
- **Package Manager** - pnpm (recommended), npm, or yarn
- **Git** - For cloning the repository

<Tip>
  We recommend using [pnpm](https://pnpm.io/) for faster installs and better disk space efficiency.
  
  Install pnpm globally:
  ```bash
  npm install -g pnpm
  ```
</Tip>

## Installation Steps

<Steps>
  <Step title="Clone the repository">
    Clone the Tendon template to your local machine:

    ```bash
    git clone https://github.com/your-org/tendon-nextjs-template.git my-app
    cd my-app
    ```

    Or use your preferred method to download the template.
  </Step>

  <Step title="Install dependencies">
    Install all required packages using your package manager of choice:

    <CodeGroup>
    ```bash pnpm
    pnpm install
    ```

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```
    </CodeGroup>

    This will install all dependencies listed in `package.json`, including:
    - Next.js and React
    - Better Auth for authentication
    - Drizzle ORM for database access
    - Radix UI components
    - Tailwind CSS and styling tools
  </Step>

  <Step title="Set up environment variables">
    Create your local environment file:

    ```bash
    cp .env.example .env.local
    ```

    <Note>
      If you already have a `.env.local` file, merge the new variables instead of overwriting it.
    </Note>

    ### Configure Required Variables

    Open `.env.local` and configure the following variables:

    ```bash .env.local
    BETTER_AUTH_URL=http://localhost:3000
    BETTER_AUTH_SECRET=change-this-in-production

    TURSO_DATABASE_URL=file:./dev.db
    TURSO_AUTH_TOKEN=
    ```

    #### Variable Details

    **BETTER_AUTH_URL**
    - Your application's base URL
    - Use `http://localhost:3000` for local development
    - Update to your production domain when deploying

    **BETTER_AUTH_SECRET**
    - A strong random secret for signing sessions and tokens
    - Generate a secure secret using:
      ```bash
      openssl rand -base64 32
      ```
    - **Critical:** Never commit this to version control

    **TURSO_DATABASE_URL**
    - Database connection string
    - For local development: `file:./dev.db` (SQLite file)
    - For production: Your Turso database URL (format: `libsql://...`)

    **TURSO_AUTH_TOKEN**
    - Leave empty for local file-based database
    - Required for remote Turso database in production

    <Warning>
      Never commit `.env.local` to version control. It's already in `.gitignore` by default.
    </Warning>
  </Step>

  <Step title="Generate authentication schema">
    Generate the Better Auth database schema:

    ```bash
    pnpm auth:generate
    ```

    This creates the authentication schema at `lib/db/schema.ts` with tables for:
    - Users
    - Sessions
    - Accounts (for OAuth providers)
    - Verification tokens

    The schema is automatically configured for SQLite compatibility.
  </Step>

  <Step title="Generate and run database migrations">
    Create migration files based on your schema:

    ```bash
    pnpm db:generate
    ```

    This analyzes your schema and generates SQL migration files in the `drizzle` directory.

    Apply the migrations to your database:

    ```bash
    pnpm db:migrate
    ```

    <Tip>
      You can run all database setup steps at once using:
      ```bash
      pnpm db:setup
      ```
      This runs `auth:generate`, `db:generate`, and `db:migrate` in sequence.
    </Tip>
  </Step>

  <Step title="Start the development server">
    Launch the development server:

    <CodeGroup>
    ```bash pnpm
    pnpm dev
    ```

    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```
    </CodeGroup>

    The application will start at [http://localhost:3000](http://localhost:3000).

    <Note>
      The template uses Vinext, a Vite-powered Next.js build tool for faster development builds and hot module replacement.
    </Note>
  </Step>
</Steps>

## Verify Installation

Once the development server is running, verify your installation:

1. **Homepage loads** - Visit `http://localhost:3000`
2. **Authentication works** - Try the auth card on the homepage to test sign up/sign in
3. **Database connected** - Check that users are being created in `dev.db`

<Tip>
  You can inspect your local database using the Drizzle Studio:
  ```bash
  pnpm drizzle-kit studio
  ```
  This opens a web UI at `https://local.drizzle.studio` to browse your database.
</Tip>

## Project Structure

After installation, your project structure will look like this:

```
my-app/
├── app/
│   ├── api/auth/[...all]/route.ts  # Better Auth API handler
│   ├── globals.css                 # Global styles
│   ├── layout.tsx                  # Root layout
│   └── page.tsx                    # Homepage
├── components/
│   └── ui/                         # UI component library (50+ components)
├── lib/
│   ├── auth.ts                     # Better Auth server config
│   ├── auth-client.ts              # Better Auth client
│   ├── db.ts                       # Drizzle database client
│   └── db/
│       └── schema.ts               # Database schema
├── drizzle/                        # Migration files
├── .env.local                      # Local environment variables
├── drizzle.config.ts               # Drizzle Kit configuration
├── tailwind.config.ts              # Tailwind CSS config
└── package.json                    # Dependencies and scripts
```

## Key Files Explained

### Authentication Configuration

**`lib/auth.ts`** - Server-side auth configuration:

```ts lib/auth.ts
import { drizzleAdapter } from "better-auth/adapters/drizzle";
import { betterAuth } from "better-auth";
import { nextCookies } from "better-auth/next-js";
import { db } from "@/lib/db";
import * as schema from "@/lib/db/schema";

const baseURL = process.env.BETTER_AUTH_URL ?? "http://localhost:3000";
const secret =
  process.env.BETTER_AUTH_SECRET ??
  (process.env.NODE_ENV === "production" ? undefined : "dev-only-secret-change-me");

if (!secret) {
  throw new Error("BETTER_AUTH_SECRET must be set in production");
}

export const auth = betterAuth({
  baseURL,
  secret,
  trustedOrigins: [baseURL],
  emailAndPassword: {
    enabled: true,
  },
  database: drizzleAdapter(db, {
    provider: "sqlite",
    schema,
  }),
  plugins: [nextCookies()],
});
```

**`lib/auth-client.ts`** - Client-side auth hooks:

```ts lib/auth-client.ts
import { createAuthClient } from "better-auth/react";

export const authClient = createAuthClient();
```

**`app/api/auth/[...all]/route.ts`** - API route handler:

```ts app/api/auth/[...all]/route.ts
import { toNextJsHandler } from "better-auth/next-js";
import { auth } from "@/lib/auth";

export const runtime = "nodejs";

export const { GET, POST } = toNextJsHandler(auth);
```

### Database Configuration

**`lib/db.ts`** - Database client setup:

```ts lib/db.ts
import { createClient } from "@libsql/client";
import { drizzle } from "drizzle-orm/libsql";

const url = process.env.TURSO_DATABASE_URL ?? "file:./dev.db";
const authToken = process.env.TURSO_AUTH_TOKEN;

const client = createClient({
  url,
  authToken,
});

export const db = drizzle(client);
```

**`drizzle.config.ts`** - Drizzle Kit configuration:

```ts drizzle.config.ts
import { defineConfig } from "drizzle-kit";

export default defineConfig({
  schema: "./lib/db/schema.ts",
  out: "./drizzle",
  dialect: "sqlite",
  dbCredentials: {
    url: process.env.TURSO_DATABASE_URL ?? "file:./dev.db",
  },
});
```

## Available Scripts

The template includes several useful scripts in `package.json`:

| Script | Description |
|--------|-------------|
| `pnpm dev` | Start development server with hot reload |
| `pnpm build` | Build production bundle |
| `pnpm start` | Start production server |

| `pnpm auth:generate` | Generate Better Auth schema |
| `pnpm db:setup` | Run all database setup steps |

## Troubleshooting

### Port 3000 Already in Use

If port 3000 is already in use, you can specify a different port:

```bash
pnpm dev -p 3001
```

Don't forget to update `BETTER_AUTH_URL` in `.env.local` to match the new port.

### Database Migration Errors

If migrations fail, try resetting your local database:

```bash
rm dev.db
pnpm db:setup
```

### Authentication Cookie Issues

Ensure that:
1. `BETTER_AUTH_URL` matches your browser URL exactly
2. Both use the same host (e.g., both use `localhost`, not mixing `localhost` and `127.0.0.1`)
3. You're not blocking cookies in your browser

### Module Not Found Errors

Clear your package manager cache and reinstall:

<CodeGroup>
```bash pnpm
rm -rf node_modules
pnpm store prune
pnpm install
```

```bash npm
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```
</CodeGroup>
