This guide will help you set up your development environment and start building with the Tendon Next.js Template.

## Prerequisites

Before you begin, make sure you have:

- Node.js 18+ installed
- pnpm (recommended) or npm/yarn
- Git
- A code editor (VS Code recommended)

## Setup Steps

<Steps>
  <Step title="Clone and Install">
    Clone the repository and install dependencies:

    <CodeGroup>
    ```bash pnpm
    git clone https://github.com/Ngineer101/tendon-nextjs-template.git my-app
    cd my-app
    pnpm install
    ```

    ```bash npm
    git clone https://github.com/Ngineer101/tendon-nextjs-template.git my-app
    cd my-app
    npm install
    ```

    ```bash yarn
    git clone https://github.com/Ngineer101/tendon-nextjs-template.git my-app
    cd my-app
    yarn install
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure Environment Variables">
    Copy the example environment file:

    ```bash
    cp .env.example .env.local
    ```

    Update `.env.local` with your configuration:

    ```env .env.local
    BETTER_AUTH_URL=http://localhost:3000
    BETTER_AUTH_SECRET=change-this-to-a-random-secret
    
    TURSO_DATABASE_URL=file:./dev.db
    TURSO_AUTH_TOKEN=
    ```

    <Warning>
    In production, always use a strong random secret for `BETTER_AUTH_SECRET`. You can generate one with:
    ```bash
    openssl rand -base64 32
    ```
    </Warning>
  </Step>

  <Step title="Setup Database">
    Generate the auth schema and run database migrations:

    ```bash
    pnpm db:setup
    ```

    This command runs three steps automatically:
    - Generates Better Auth schema at `lib/db/schema.ts`
    - Generates SQL migrations
    - Applies migrations to your local database
  </Step>

  <Step title="Start Development Server">
    Launch the development server:

    ```bash
    pnpm dev
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser to see your app!

    <Tip>
    For reliable cookie/session behavior, make sure `BETTER_AUTH_URL` matches your browser URL (both should use `localhost`).
    </Tip>
  </Step>
</Steps>

## Quick Examples

### Using Authentication

```tsx Client Component
"use client";

import { authClient } from "@/lib/auth-client";

export function AuthExample() {
  const { data: session } = authClient.useSession();

  const handleSignIn = async () => {
    await authClient.signIn.email({
      email: "user@example.com",
      password: "securepassword",
    });
  };

  if (session) {
    return <div>Welcome, {session.user.email}!</div>;
  }

  return <button onClick={handleSignIn}>Sign In</button>;
}
```

### Querying the Database

```tsx Server Component
import { db } from "@/lib/db";
import { user } from "@/lib/db/schema";

export async function UserList() {
  const users = await db.select().from(user);

  return (
    <ul>
      {users.map((u) => (
        <li key={u.id}>{u.email}</li>
      ))}
    </ul>
  );
}
```

## Troubleshooting

<Accordion title="Database migrations failing">
  Make sure you've run `pnpm auth:generate` first to create the schema file. If issues persist, delete `dev.db` and `drizzle/` folder, then run `pnpm db:setup` again.
</Accordion>

<Accordion title="Authentication not working">
  Verify that:
  - `BETTER_AUTH_URL` matches your browser URL exactly
  - `BETTER_AUTH_SECRET` is set
  - The auth API route at `app/api/auth/[...all]/route.ts` exists
  - You've run database migrations
</Accordion>

<Accordion title="Port 3000 already in use">
  Either stop the process using port 3000, or run on a different port:
  ```bash
  PORT=3001 pnpm dev
  ```
  Remember to update `BETTER_AUTH_URL` to match the new port.
</Accordion>
